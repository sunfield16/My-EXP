* 行ロック: テーブル内の特定の行だけをロックする
    - 範囲が狭いので他のクエリを妨げにくい
    - ロックする側のクエリでインデックスが効いてないと使用不可
* テーブルロック: テーブルの一部や全体をロックする
    - 範囲が広いので他のクエリにも影響しやすい

## 共有・排他
* 共有ロック: 他から書き込みできないが読み取りはできる
* 排他ロック: 他から書き込みも読み取りも不可
