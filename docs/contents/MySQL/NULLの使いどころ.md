1つの指針として、MySQLのNull（Null許容）をいつ使うのか考える。

## 意味合いで考える
* NULL
    - 値が**存在しない**
    - 値が**定義されてない**
* NOT NULL（空文字や0など指定）
    - 「ないことを表す値」を**設定する**

## 仕様で考える
* 値がなくても問題にならない→NULL許容？
    - オプション的なもの
- フォームなどで「未入力」や「指定なし」が許容される→NULL許容？
    - アンケートの自由入力などは空文字をデフォルトにしそうな感じ
* 初期値が「空文字」や「0」など明確に存在している→NOT NULL
    - 値があることが前提になっている
    - 例：クエストのクリア回数をカウントするカラム（最初は必ず0になる）
* Enumなどで「未設定」という値が存在する→NOT NULL
    - 「未設定」も数値として扱った方がソース上などで都合がよいと思う
